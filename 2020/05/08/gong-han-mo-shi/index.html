工厂模式学习笔记

介绍

​ 工厂模式有简单工厂模式，工厂方法模式和抽象工厂模式，都属于创建型模式；主要功能是帮助我们把对象的实例化部分抽取出来，以便降低系统中代码耦合度，并增强系统的扩展性。

本质

  • 实例化对象不使用new，用工厂方法代替
  • 将选择实现类，创建对象统一管理和控制；从而将调用者跟我们的实现类解耦

简单工厂模式

实现

  1. 创建一个接口

    package bml;

public interface Car {
    /**
     * 输出车名
     */
    void name();
}

  1. 编写两个实现类

    package bml;

public class Wuling implements Car{
    @Override
    public void name() {
        System.out.println("五菱宏光");
    }
}
    package bml;

public class Tesila implements Car {
    @Override
    public void name() {
        System.out.println("特斯拉");
    }
}

  1. 创建一个工厂,根据提供的消息返回特定的对象

    package bml;

public class CarFactory {
    /**
     * 创建一个工厂，生成基于给定信息的实体类的对象
     * 此处使用静态方法 其它类就可以直接使用 类名.方法 直接调用
     * 如果不加 static 则需要先创建该工厂对象 再调用对象的方法
     */
    public static Car get(String car) {
        if ("五菱".equals(car)) {
            return new Wuling();
        } else if ("特斯拉".equals(car)) {
            return new Tesila();
        } else {
            return null;
        }
    }
}

  2. 使用工厂，通过传递类型信息来获取实体类的对象

    package bml;

public class Customer {
    public static void main(String[] args) {
        Car car1 = CarFactory.get("五菱");
        Car car2 = CarFactory.get("特斯拉");
        car1.name();
        car2.name();
    }
}

小结

  • 该模式优点在于实现对象的创建和对象的使用分离，将对象的创建交给专门的工厂类负责
  • 其缺点是工厂类不够灵活，增加新的具体产品就需要修改工厂类的判断逻辑代码（不符合开闭原则），而且产品较多时，工厂方法代码逻辑将会非常复杂

开闭原则

一句话：软件对象（类、模块、方法等）应该对于扩展开放，对修改关闭。

工厂方法模式

实现

  1. 创建一个接口

    package MethodFactory;

public interface Car {
    /**
     * 输出车名
     */
    void name();
}

  1. 编写实现类

    package MethodFactory;

public class Tesila implements Car {
    @Override
    public void name() {
        System.out.println("特斯拉车");
    }
}

  1. 创建一个工厂接口

    package MethodFactory;

public interface CarFactory {
    Car getCar();
}

  1. 编写实现类

    package MethodFactory;

public class TesilaFactory implements CarFactory{
    @Override
    public Car getCar() {
        return new Tesila();
    }
}

  1. 使用工厂

    package MethodFactory;

public class Custormer {
    public static void main(String[] args) {
        Car car = new WulingFactory().getCar();
        car.name();
    }
}

  1. 扩展如下

    package MethodFactory;

// 1.新增汽车实现汽车接口
public class Dazhong implements Car{
    @Override
    public void name() {
        System.out.println("大众汽车");
    }
}
    package MethodFactory;

// 2.新增汽车工厂实现汽车工厂接口
public class DazhongFactory implements CarFactory{
    @Override
    public Car getCar() {
        return new Dazhong();
    }
}
    package MethodFactory;

public class Custormer {
    public static void main(String[] args) {
        Car car = new WulingFactory().getCar();
        car.name();
        // 3.使用新增的汽车工厂创建对象
        Car car2 = new DazhongFactory().getCar();
        car2.name();
    }
}

小结

​ 此模式中，通过定义一个抽象的核心工厂类，并定义创建产品对象的接口，创建具体产品实例的工作延迟到其工厂子类去完成。

优点

  • 核心类只关注工厂类的接口定义，而具体的产品实例交给具体的工厂子类去创建。
  • 当系统需新增一个产品时，无需修改原有代码，只需添加一个具体产品类和其对应的工厂子类即可。
  • 增强系统的扩展性，符合面向对象编程的开闭原则。
