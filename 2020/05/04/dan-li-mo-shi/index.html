单例模式学习笔记

概念

所谓类的单例模式，就是采取一定的方法在整个软件系统中，对某个类只能存在一个实例，并且该类只提供一个取得其对象实例的方法（静态方法）

特点

  • 类构造器私有
  • 持有自己类型的属性
  • 对外提供获取实例的静态方法

方式

  1. 饿汉式（静态常量）
  2. 饿汉式（静态代码块）
  3. 懒汉式（线程不安全）
  4. 懒汉式（线程安全，同步方法）
  5. 懒汉式（线程安全，同步代码块）
  6. 双重检查
  7. 静态内部类
  8. 枚举

饿汉式（静态常量）

package bml;

/**
 * 饿汉式 静态常量
 */
public class Hungry {

    /**
     * 1.构造器私有化,则不能通过New关键字来创建对象
     */
    private Hungry() {

    }

    /**
     * 2.本类内部创建对象实例
     */
    private static Hungry INSTANCE = new Hungry();

    /**
     * 3.提供一个公有的静态方法，返回实例对象
     */
    public static Hungry getInstance() {
        return INSTANCE;
    }

    public static void main(String[] args) {
        Hungry instance = Hungry.getInstance();
        Hungry instance1 = Hungry.getInstance();
        System.out.println(instance.hashCode());
        System.out.println(instance1.hashCode());
    }
}

优点：写法简单，在类装载的时候就完成了实例化，避免了线程同步问题。

缺点：在类装载的时候就完成实例化，无法达到Lazy Loading效果；如果从始至终就木有用过这个类，就会造成内存资源的浪费。

饿汉式（静态代码块）

package bml;

/**
 * * 饿汉式 静态代码块
 */
public class HungryStatic {

    /**
     * 1.构造器私有
     */
    private HungryStatic() {

    }

    /**
     * 2.静态代码块 创建实例
     */
    private static HungryStatic INSTANCE;
    static {
        INSTANCE = new HungryStatic();
    }

    /**
     * 3.返回实例
     */
    public static HungryStatic getInstance() {
        return INSTANCE;
    }

    public static void main(String[] args) {
        HungryStatic instance1 = HungryStatic.getInstance();
        HungryStatic instance2 = HungryStatic.getInstance();
        System.out.println(instance1.hashCode());
        System.out.println(instance2.hashCode());
    }
}

优缺点和静态常量一致。

懒汉式（线程不安全）

package bml;

/**
 * 懒汉式 线程不完全
 */
public class LazyOne {

    private LazyOne() {}

    private static LazyOne lazy;

    /**
     * 提供静态的公有方法
     * 当使用到该方法时才去创建该实例
     */
    public static LazyOne getInstance() {
        if (lazy == null) {
            lazy = new LazyOne();
        }
        return lazy;
    }

    public static void main(String[] args) {
        LazyOne instance1 = LazyOne.getInstance();
        LazyOne instance2 = LazyOne.getInstance();
        System.out.println(instance1.hashCode());
        System.out.println(instance2.hashCode());
    }
}

优点：起到了懒加载效果，但只在单线程下有效

缺点：如果在多线程下，线程A通过了非空判断，还未来得及创建实例，这时线程B也通过了非空判断，也要创建实例；这时就会产生多个实例。

结论：在实际开发中不要使用这种方式。

懒汉式（线程安全 同步方法）

package bml;

/**
 * 懒汉式 线程安全 同步方法
 */
public class LazyTwo {

    private LazyTwo() {}

    private static LazyTwo lazy;

    /**
     * 加入同步代码 解决线程不安全问题
     * 在第一种的基础上加上 synchronized 关键字
     */
    public static synchronized LazyTwo getInstance() {
        if (lazy == null) {
            lazy = new LazyTwo();
        }
        return lazy;
    }
}

优点：通过同步以解决线程不安全问题

缺点：效率低下，每一个线程想获得该类的实例都要进行同步操作，其实getInstance()方法只执行一次实例化就够了，如果后面的线程想获得该实例，可以直接return返回。

结论：实际开发中，不推荐这种方式。

双重检查

package bml;

public class DoubleCheck {

    private DoubleCheck(){}

    /**
     * volatile作用
     * 一旦实例改变就立刻同步到主存中
     */
    private static volatile DoubleCheck INSTANCE;

    public static DoubleCheck getInstance() {
        // 假如两个线程执行到这里
        if (INSTANCE == null) {
            // 线程A，B都通过了非空判断，但只能有一个执行下列代码
            // 比如A进去执行，通过非空判断，创建实例后返回
            // B接着进去后就无法通过非空判断，只能退出,从而保证了对象的单例
            synchronized (DoubleCheck.class) {
                if (INSTANCE == null) {
                    INSTANCE = new DoubleCheck();
                }
            }
        }
        return INSTANCE;
    }
}

优点：解决线程安全问题、解决懒加载问题；同时保证了效率，推荐使用。

静态内部类

package bml;

/**
 * 静态内部类
 */
public class StaticInner {

    private StaticInner() {}

    private static class StaticInner1 {
        private static final StaticInner INSTANCE = new StaticInner();
    }

    public static final StaticInner getInstance() {
        return StaticInner1.INSTANCE;
    }
}

特点：1.当主类在类装载的时候，其内部类是不会被装载的（保证了懒加载效果）

​ 2.当内部类装载的时候，线程是安全的，而且只会装载一次;推荐使用

Final与Static的前后顺序对整个类并无影响。

枚举

package bml;

public enum  EnumTest {

    INSTANCE;

    public void test() {
        System.out.println("123456");
    }

    public static void main(String[] args) {
        EnumTest instance1 = EnumTest.INSTANCE;
        EnumTest instance2 = EnumTest.INSTANCE;
        System.out.println(instance1.hashCode());
        System.out.println(instance2.hashCode());
        instance1.test();
        instance2.test();
    }
}

优点：1.避免多线程同步问题

​ 2.防止反序列化重新创建新的对象；推荐使用
